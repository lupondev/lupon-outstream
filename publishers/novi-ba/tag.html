AbortError Fix — Prebid HB skripte

Problem: adxbid.info HB skripte (novi_ba_inArticle.js, midArticle.js, underArticle.js) abortuju fetch requestove 10+ puta. Uzrok je ili Prebid timeout pre-empts requests ili AbortController koji se triggera prerano.

Rješenje: povećaj Prebid bidder timeout na 2000ms i dodaj AbortController cleanup.

Dodaj ovo u Prebid config prije pbjs.requestBids poziva:

<script> // Prebid config fix — povećaj timeout, spriječi AbortError var pbjs = pbjs || {}; pbjs.que = pbjs.que || []; pbjs.que.push(function() { pbjs.setConfig({ bidderTimeout: 2000, // Povećaj sa default 1000ms enableSendAllBids: false, // Spriječi race condition između HB i outstream playera cache: { url: 'https://prebid.adnxs.com/pbc/v1/cache' } }); }); </script>
!

Dijagnoza i redosljed deploymenta

Root cause chain:
1. attachTo: {} → IntersectionObserver se ne registrira → VAST request nikad ne ide → player zauvijek u pending stanju
2. defer na apex.js → race condition → MagniteApex nije spreman kad renderAd() se poziva
3. Adform height:0; overflow:hidden → inview.js ne može kreirati player
4. AbortError u HB skriptama → Prebid aukcija ne završava → bidi se ne sakupljaju

Deploy redosljed: Fix 1 (attachTo) → Fix 2 (defer) → Fix 3 (adform CSS) → Fix 4 (Prebid timeout)

Test: Nakon deploymenta, otvori novi.ba članak i provjeri Network tab → trebao bi vidjeti GET request prema optimized-by.rubiconproject.com/a/api/vast
